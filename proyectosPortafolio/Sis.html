ERP Empresarial

1. Visión General

Aplicación de ejemplo para enseñar los fundamentos de un sistema ERP integral con:

  1. Gestión de Inventarios
  2. Facturación y Ventas
  3. Recursos Humanos
  4. Reportes Analíticos Avanzados
2. Interfaz de Usuario: Angular

¿Qué es? Framework de Google para SPAs.

Conceptos clave:

  1. Rutas y Lazy Loading para módulos de inventario, facturación, RRHH y reportes
  2. Forms reactivos para creación y edición de registros
  3. Servicios HTTP para consumir APIs

Ejemplo didáctico:

// inventory-list.component.ts
export class InventoryListComponent {
  items$: Observable;
  constructor(private inventoryService: InventoryService) {
    this.items$ = this.inventoryService.getAll();
  }
}
3. Servidor y Lógica de Negocio: NestJS

¿Qué es? Framework Node.js con TypeScript.

Conceptos clave:

  1. Módulos separados: InventoryModule, BillingModule, HrModule, ReportsModule
  2. Guards y interceptors para seguridad y logging
  3. Servicios transaccionales con Unit of Work

Ejemplo didáctico:

// billing.controller.ts
@Controller('billing')
export class BillingController {
  constructor(private billingService: BillingService) {}
  @Post('invoice')
  create(@Body() dto: CreateInvoiceDto) {
    return this.billingService.generateInvoice(dto);
  }
}
4. Almacenamiento de Datos: PostgreSQL

¿Qué es? Base de datos relacional avanzada.

Conceptos clave:

  1. Relaciones entre productos, pedidos, empleados y nóminas
  2. Transacciones complejas para facturación segura
  3. Vistas y funciones para reportes analíticos

Ejemplo didáctico:

-- schema.sql
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  stock INT DEFAULT 0
);
CREATE TABLE invoices (
  id SERIAL PRIMARY KEY,
  total NUMERIC(10,2) NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);
CREATE TABLE employees (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);
CREATE VIEW sales_summary AS
 SELECT date_trunc('day', created_at) AS day,
        SUM(total) AS daily_total
 FROM invoices
 GROUP BY day;
5. Reportes Analíticos: Charts con Angular

¿Qué es? Visualización de datos en front-end.

Conceptos clave:

  1. Uso de librerías como Chart.js o ngx-charts
  2. Endpoints en NestJS que devuelven datos agregados
  3. Componentes Angular para mostrar gráficas de ventas, inventario y nómina

Ejemplo didáctico:

// report.component.ts
this.http.get('/api/reports/sales').subscribe(data => {
new Chart('canvas', {
    type: 'bar',
    data: {
    labels: data.map(d => d.day),
    datasets: [{ data: data.map(d => d.daily_total), label: 'Ventas Diarias' }]
    }
 });
});
6. Flujo de Trabajo Sugerido
  1. Setup: ng new erp-ui, nest new erp-api, configurar PostgreSQL.
  2. Módulos CRUD: Implementar inventarios y facturación primero.
  3. Recursos Humanos: CRUD de empleados y nómina.
  4. Reportes: Crear vistas en BD y endpoints en NestJS.
  5. Dashboards: Integrar gráficos en Angular.
  6. Pruebas y Documentación: README con diagramas ER y rutas API.
